# Custom state change

The custom state change alert triggers when an object moves into or out of a state you define. Use it to monitor conditions that the built-in alert types don't cover.

## How it works

You define a state for an object category and the conditions that represent it. Lumana monitors the camera feed and triggers the alert when the object's state changes, capturing a clip of the transition.

## When to use it

This alert is useful when you need to track a specific condition tied to your operation rather than a generic detection type.

- Detecting when a defined object type enters or leaves a monitored zone.
- Monitoring asset or equipment states unique to your workflow.
- Triggering alerts based on custom object classifications not covered by standard alert types.

These are the most common starting points, but the alert can be adapted to any condition that maps to a detectable object state in your environment.
