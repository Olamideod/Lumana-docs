# People alerts

Lumana can detect and track people across your cameras and trigger alerts based on their behavior, identity, or location. Person-specific alerts span three categories depending on what you want to monitor.

## Alert types

The alerts below are organized by category. Choose the one that matches what you're trying to detect.

### Security

Security alerts trigger based on a person's behavior or presence in a specific location.

- **Motion**: Triggers when a person moves within a defined zone or across the camera view.
- **Proximity**: Triggers when a person stays within a defined distance of another person or object for a set duration.
- **Trespassing**: Triggers when a person appears on camera in a restricted area at any time or during defined hours.
- **Zone protection**: Triggers when a person enters a defined zone and remains for a specified duration.
- **Loitering**: Triggers when a person stays in a zone longer than a configured time threshold.

### Identification

Identification alerts trigger when Lumana recognizes a specific person based on your organization's database.

- **Face recognition**: Triggers when a person appears on, or does not appear on, a list of known faces.

### Tracking

Tracking alerts monitor how a person moves relative to defined zones and boundaries.

- **Appearing**: Triggers when a person enters a defined zone.
- **Disappearing**: Triggers when a person leaves a defined zone.
- **Line crossing**: Triggers when a person crosses a defined line.
- **Occupancy**: Triggers when the number of people in a zone reaches a configured threshold.

Select the category that matches the behavior you want to detect, then follow the steps below.

## Configure a people alert

The steps are the same regardless of which alert type you choose. When prompted to select an object type, select **Person**.

1. Select **Alerts** in the left navigation bar, then select **Configurations**.
2. Select **Add alert**.
3. Select the relevant category: **Security**, **Identification**, or **Tracking**.
4. Select an alert type and follow the configuration steps.
5. When prompted to select an object type, choose **Person**.
