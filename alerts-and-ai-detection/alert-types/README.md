# Alert types

Lumana's alert types are organized into five categories based on what they detect and how they're triggered. Each type is documented separately with a description, recommended use cases, and configuration steps where available.

## Security

Alerts that trigger based on behavior, presence, or events that pose a direct security risk.

- [Motion](motion.md)
- [Tampering](tampering.md)
- [Proximity](proximity.md)
- [Trespassing](trespassing.md)
- [Zone protection](zone-protection.md)
- [Tailgating](tailgating.md)
- [Gun drawn](gun-drawn.md)
- [Gun detection](gun-detection.md)
- [Violence](violence.md)
- [Speed control](speed-control.md)
- [Door state change](doors.md)
- [Fire](fire.md)

## Identification

Alerts that trigger based on the identity of a person or vehicle.

- [Suspect alerts](suspect-alerts.md) — face recognition and hands detected
- [License plate recognition](license-plate.md)
- [Container detection](container.md)
- [Missing object](missing-object.md)

## Tracking

Alerts that monitor how objects move relative to defined zones, boundaries, and paths.

- [Appearing](appearing.md)
- [Disappearing](disappearing.md)
- [Loitering](loitering.md)
- [Line crossing](line-crossing.md)
- [Occupancy](occupancy.md)
- [Traffic control](traffic-control.md)
- [Absence](absence.md)

## Safety and compliance

Alerts that detect unsafe behavior or missing protective equipment.

- [Fall](fall.md)
- [Smoking and vaping](smoking-and-vaping.md)
- [Protective gear](protective-gear.md)
- [Personal safety](personal-safety.md)
- [Gloves](gloves.md)

## Status

Alerts that notify you when hardware goes offline or reconnects.

- [Device status](device-status.md)
